# 导出明细、TI2I 坏例双图与 Prompt 预览修复设计

## 背景

当前有三处行为不符合预期：导出的 Overall Sheet 有统计内容，但已评测记录没有进入对应的明细 Sheet；看板 TI2I 坏例高清预览只显示当前模型结果图；评测页和看板的高清预览均未在预览顶部显示 Prompt。

本次采用最小修复，不重构导出格式或预览控制器，不新增依赖。

## 目标行为

### 导出明细

- 保留现有 `Overall + 各明细 Sheet` 的工作簿结构、Sheet 命名和列定义。
- 已通过导出全局筛选且实际参加评测的记录应写入其对应的明细 Sheet，不能因为没有命中当前细分维度行集合而只留下表头。
- 不新增“整体结果”列，不把 Overall 判定复制或伪装成细分维度判定。
- 已有维度判定仍按现有三列格式输出；真实不存在的维度判定保持为空。
- 结果筛选、坏例筛选、时间、场景、评测人和评测模式语义保持不变。

### TI2I 坏例高清预览

- T2I 坏例继续显示当前坏例结果图。
- TI2I 坏例显示两张图：参考图和当前坏例所属模型的结果图。
- 双图复用现有高清预览的缩放、平移、同步、放大镜和按住比较能力。
- 不显示另一个模型的结果图，避免把坏例目标混淆为普通 A/B 明细预览。
- 参考图缺失时安全退化为当前结果单图预览。

### 高清预览 Prompt

- 评测页高清预览显示当前任务 Prompt。
- 看板评测明细与坏例高清预览显示对应记录 Prompt。
- Prompt 位于高清预览顶部标题区，可换行；图片内容区从标题区下方开始，文字不得覆盖图像。
- 缺少 Prompt 时不显示空占位文案；关闭或打开下一条预览时不得残留上一条 Prompt。
- Prompt 通过文本节点或 `textContent` 写入，不作为 HTML 解析。

## 实现边界

- 导出修复限定在 `app_core/export_service.py` 的明细行集合构造及其单元测试。
- 看板修复限定在 `templates/dashboard.html` 的坏例 payload、预览标准化、顶部布局和相应 UI 测试。
- 评测页修复限定在 `templates/index.html` 的 lightbox 标题、Prompt 同步和相应 UI 测试。
- 保留现有 PreviewController、图片接口、坏例查询接口、导出接口和筛选表单。
- 不修改数据库数据，不生成缺失的细分维度结果。

## 数据流

1. 导出服务先应用现有全局筛选得到 Overall 行，并分别计算所选维度的匹配集合；明细 Sheet 使用与该 Sheet 对应且已评测的记录集合，维度标记仍由真实维度匹配集合决定。
2. 看板坏例行已经携带 `prompt` 和 `ref_img`；点击时将它们与当前模型结果图组成专用坏例预览 payload，再由统一标准化函数生成单图或双图 pane。
3. 看板普通明细 payload 同样携带 `prompt`；打开预览时写入顶部 Prompt 节点。
4. 评测页打开 lightbox 时从 `state.currentTask.prompt` 同步顶部 Prompt 节点。

## 错误与降级

- 参考图 URL 为空时坏例预览只渲染结果图，且不展示无意义的同步或跨图比较控件。
- Prompt 为空时顶部 Prompt 节点隐藏，图片区域正常占满剩余空间。
- 图片加载失败沿用现有失败状态，不影响另一张图片及关闭操作。
- 导出只输出数据库中真实存在的评测值，空维度继续为空。

## 测试与验收

- 新增导出回归测试：构造 Overall 可见但旧明细集合遗漏的已评测记录，先验证 Sheet 只有表头，再修复为包含明细行；同时断言未评测维度仍为空。
- 新增看板运行时测试：TI2I 坏例 payload 标准化为参考图和当前结果图，T2I 与缺参考图场景退化为单图。
- 新增看板测试：普通明细和坏例都把 Prompt 传入预览，顶部 Prompt 使用文本写入并在打开下一条时刷新。
- 新增评测页测试：lightbox 顶部显示当前任务 Prompt，并验证布局把标题/Prompt 与图片网格放在不同网格行。
- 运行相关专项测试、完整测试、Python 编译、模板 JavaScript 语法检查和 `git diff --check`。

## 非目标

- 不重命名或重排现有 Sheet。
- 不为 Overall-only 数据虚构细分维度结果。
- 不重设计高清预览工具栏或主评测界面 Prompt 卡片。
- 不改变坏例筛选、分页或统计。
