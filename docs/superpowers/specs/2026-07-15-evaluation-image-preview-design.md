# 评测图片预览优化设计

## 目标

优化评测页中参考图（如有）、候选图 A 和候选图 B 的显示空间与一致性，并在主评测页和高清预览中提供同步对比、放大镜及常用图片检查工具。

## 范围

- 调整 T2I 两图与 TI2I 三图的主页面布局。
- 重构现有高清预览，使其复用统一的图片预览控制逻辑。
- 新增桌面端右侧悬浮、窄屏底部横向的可折叠工具栏。
- 不修改评测任务、投票、坏例标签和后端数据接口。

## 架构

新增共享 `PreviewController`，统一管理主页面与高清预览中的图片交互。每个页面区域注册为独立预览组，但使用一致的状态结构和事件处理方式。

每个预览组管理：

- 当前显示模式：适应窗口、适应宽度、适应高度或实际尺寸。
- 当前缩放比例。
- 归一化中心点，即图片坐标中的百分比位置。
- 同步锁定状态，默认开启。
- 放大镜、背景主题和临时拖拽状态。

同步平移不直接复制像素偏移，而是同步归一化中心点。这样即使 Ref、A、B 的原始分辨率不同，也能观察各图中的对应位置。

## 主页面布局

- 放宽主内容区的最大宽度，减少卡片内边距，使图片尽量使用浏览器可用空间。
- T2I 使用两列等宽布局，TI2I 使用三列等宽布局。
- 同一预览组中的图片舞台采用统一高度，并根据视口高度动态计算。
- 图片保持原始宽高比，默认完整显示，不裁切、不拉伸。
- 窄屏下改为单列纵向布局，工具栏移动至底部。
- Ref、A、B 均可打开高清预览，避免参考图无法直接检查。

## 高清预览布局

- 弹窗扩展为接近全浏览器视口的沉浸式预览区域。
- T2I 两图等宽，TI2I 三图等宽。
- 工具栏悬浮覆盖，不永久占用图片宽度。
- 打开时默认使用适应窗口模式、居中位置，并保持同步锁定开启。
- 关闭时清理拖拽、指针捕获和放大镜等临时状态。

## 工具栏

主页面与高清预览提供同一组核心工具：

1. **同步锁定**：默认开启；缩放、平移和定位同步到组内全部图片。
2. **显示模式**：适应窗口、适应宽度、适应高度和 `1:1` 实际尺寸。
3. **缩放**：按钮、鼠标滚轮和 `+`、`-` 快捷键；缩放中心为当前鼠标位置。
4. **平移**：直接拖动图片；按住空格可临时进入平移模式。
5. **放大镜**：显示跟随鼠标的矩形镜框，默认约 `3×` 放大局部细节。
6. **复位**：恢复当前组的默认显示模式与居中位置，不修改同步锁定状态。
7. **全屏预览**：从主页面进入高清预览。
8. **背景切换**：在明暗背景间切换，方便检查透明边缘、暗部或亮部细节。
9. **图片信息**：显示原始尺寸、当前缩放比例和当前显示模式。
10. **快捷键帮助**：展示可用的预览快捷键，同时避免覆盖现有评分快捷键。

## 放大镜规则

- 开启后，鼠标进入图片舞台时显示固定尺寸的矩形镜框。
- 镜框内容来自原图对应区域，使用独立的局部放大层，不改变主图片缩放状态。
- 鼠标离开舞台、切换任务或关闭预览时隐藏镜框。
- 同步锁定开启时，在其他图片上显示对应位置标记；只在当前悬停图片上显示实际放大镜，避免多个镜框遮挡画面。
- 触屏设备不启用悬停放大镜，但保留缩放与拖拽。

## 状态与数据流

1. 任务加载完成后渲染图片舞台并等待图片尺寸可用。
2. `PreviewController` 根据容器与自然尺寸计算各显示模式的基础比例。
3. 用户在任一图片中缩放或平移时，控制器更新该预览组的比例和归一化中心点。
4. 同步开启时，控制器将状态映射到组内全部图片；关闭时只更新当前图片。
5. 窗口尺寸变化时重新计算基础比例，并保留当前归一化观察中心。
6. 加载下一任务时清理上一任务状态，恢复默认显示模式与同步锁定。

## 错误与边界处理

- 图片加载中显示明确的加载状态。
- 图片加载失败时显示占位和错误提示，并从同步计算中排除该图片。
- 缩放设置合理上下限，防止图片过小或超大导致页面失控。
- 平移范围允许查看边缘，但限制图片完全移出舞台。
- `Escape` 仅关闭高清预览；预览快捷键在输入控件聚焦时不触发。
- 所有预览操作与投票、提交和坏例选择状态完全隔离。

## 响应式规则

- 宽屏：右侧悬浮可折叠工具栏，图片保持多列。
- 中等宽度：缩小工具栏按钮标签，仍优先保持多列。
- 窄屏：图片改为单列，工具栏改为底部横向可滚动区域。
- 动态使用视口高度计算图片舞台，避免固定高度在小屏或大屏下浪费空间。

## 测试方案

### 自动测试

- 扩展前端契约测试，检查主页面和高清预览都包含共享工具栏入口。
- 检查同步锁定默认开启。
- 检查四种显示模式、放大镜、复位、背景切换和图片信息控件存在。
- 检查 Ref 图片与候选图一样可以进入高清预览。
- 检查预览快捷键不会替换现有 `1 / 2 / 3 / Enter` 评分快捷键。

### 手动验证

- 分别验证 T2I 两图和 TI2I 三图布局。
- 验证相同与不同原始分辨率图片的同步缩放和平移。
- 验证同步关闭后仅操作当前图片。
- 验证放大镜位置、边缘行为及对应位置标记。
- 验证窗口缩放、窄屏布局和工具栏折叠。
- 验证任务切换、图片加载失败、弹窗开关和快捷键。

## 完成标准

- Ref、A、B 在同一预览组中具有一致的舞台尺寸，并尽量填满可用窗口。
- 主页面和高清预览均支持默认开启的同步缩放与平移。
- 放大镜可以稳定检查局部细节且不改变主预览状态。
- 所有新增工具在桌面和窄屏布局下可用。
- 现有评测流程、投票快捷键和后端接口保持兼容。
