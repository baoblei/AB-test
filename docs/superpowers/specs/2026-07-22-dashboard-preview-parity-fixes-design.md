# 看板高清预览与评测界面对齐修复设计

## 背景

看板高清预览已经具备多图缩放、按住对比和放大镜，但与评测界面仍有三处可见差异：TI2I 跨越两张图的比较按钮使用文本折线符号；右侧图片信息默认常显并覆盖图片；同步放大镜只在当前图显示镜片，其他图片仅显示定位点。

本次只修复这三处差异，不修改后端接口、看板筛选/分页/统计/导出、坏例单图边界或评测页面行为。

## 目标行为

### 三图跨图对比图标

- 相邻图片之间的左右箭头保持现状。
- TI2I 两个跨越非相邻图片的按钮使用与评测界面相同的 SVG 折线路径，视觉形状为 `^`，并带目标方向箭头。
- 按住、键盘 Space/Enter、释放和覆盖比较的数据语义保持不变。

### 右侧工具栏和文字面板

- 桌面视口默认只显示窄图标栏，图片信息和快捷键文字均隐藏。
- 点击 `?` 后工具栏展开为 220px，显示快捷键说明和当前活动图片信息；再次点击恢复窄栏。
- 展开时图片网格同步预留右侧空间，文字面板不得覆盖图片。
- 低高度桌面和不超过 760px 的窄屏继续使用底部横向滚动工具栏，所有按钮及文字内容均可滚动到达。
- 坏例单图仍不显示同步按钮或跨图比较控件。

### 同步放大镜

- 同步开启时，在任一图片上移动鼠标，所有已加载且未失败的 pane 都在相同归一化坐标显示完整圆形放大镜。
- 每个镜片使用所在 pane 自己的原图作为背景，并使用与评测界面一致的三倍放大倍率和边界夹取规则。
- 同步关闭时只显示当前来源 pane 的镜片，其他 pane 不显示镜片或蓝色定位点。
- 指针离开实际图片、关闭放大镜、关闭预览、图片失败或重新渲染时，清理所有镜片状态。

## 实现边界

- 在 `templates/dashboard.html` 内复用 `templates/index.html` 已验证的折线 SVG、工具栏展开状态和同步镜片坐标算法。
- 不抽取跨模板公共组件，不新增第三方依赖。
- 保留当前 `PreviewController`、明细 T2I A/B、TI2I Ref/A/B 和坏例单图的数据边界。
- 保留缩放范围 `0.1–12`、图片等比显示、同步缩放和平移行为。

## 测试与验收

- 单元/运行时测试验证 folded compare 输出 SVG polyline，不再输出文本 `└←` 或 `→┘`。
- 真实 Chrome 几何测试验证桌面默认文字隐藏、点击帮助后展开 220px、图片网格预留空间且无重叠，并验证低高度/窄屏工具仍可达。
- 运行时测试验证同步开启时 Ref/A/B 三个镜片同时可见、分别使用各自图片背景；同步关闭时只有来源镜片可见。
- 浏览器人工复验截图中的 TI2I 三图场景，并运行专项测试、全量测试、JavaScript 语法检查和 `git diff --check`。

## 非目标

- 不改变按住比较的覆盖算法或按钮数量。
- 不重设计评测界面。
- 不新增移动端手势、全屏模式或后端图片处理。
