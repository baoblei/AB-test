# 看板高清图片预览设计

## 目标

将评测看板中明细和坏例表格的高清图片预览升级为与评测界面一致的沉浸式检查体验。明细预览继续显示完整对比组；坏例预览保持单图，只提供图片检查工具，不补齐其他模型或参考图。

## 范围

- 升级看板明细表格中图片缩略图打开的高清预览。
- 升级看板坏例表格中图片缩略图打开的高清预览。
- 复用评测界面已有的缩放、平移、显示模式、放大镜、背景、复位和覆盖对比交互模型。
- 不修改明细、坏例或图片接口，不改变看板筛选、统计、导出和评测流程。
- 不在本次改动中抽取跨模板公共静态前端组件。

## 方案

在 `templates/dashboard.html` 内实现与评测界面 `PreviewController` 行为兼容的看板预览控制层。该方案复用成熟的状态结构和交互规则，同时把改动限制在看板模板及其测试中，避免为了单一需求重构两个大型内嵌模板。

不采用仅增加滚轮缩放的简化方案，因为它无法提供与评测高清预览一致的显示模式、平移、放大镜和覆盖对比能力。也暂不将两个模板的预览代码整体抽成公共静态组件，因为当前项目没有通用前端资源层，这会显著扩大回归范围。

## 预览类型

### 明细预览

- T2I 显示候选图 A 和候选图 B。
- TI2I 在参考图存在时显示参考图、候选图 A 和候选图 B；参考图缺失时退化为 A/B 两图。
- 默认开启同步缩放和平移。
- 双图和三图均提供按住覆盖对比按钮，方向和布局规则与评测界面一致。
- 图片标签使用实际模型名称；参考图标签固定为“参考图”。

### 坏例预览

- 只显示当前坏例记录中被点击模型的图片。
- 不自动补齐参考图、另一模型图片或完整 A/B 组合。
- 不显示同步开关和图间覆盖对比按钮。
- 仍提供缩放、平移、适应窗口、适应宽度、适应高度、1:1、放大镜、背景切换、复位、图片信息和快捷键帮助。

## 布局与视觉

- 高清预览覆盖层使用接近整个浏览器视口的沉浸式深色舞台。
- 顶部显示“高清预览”标题和关闭按钮，图片检查工具悬浮在舞台边缘。
- T2I 两图和 TI2I 三图使用等宽列；坏例单图在可用空间内居中显示。
- 各图片使用独立舞台容器，保持原始宽高比，不裁切、不拉伸。
- 窄屏时图片纵向排列，工具栏改为底部横向可滚动布局。
- 图片加载中显示加载状态，加载失败时显示可读错误提示并停止对该图片执行同步变换。

## 状态与交互

看板预览控制层为当前打开的覆盖层维护一个预览组。每个组包含：

- 显示模式：适应窗口、适应宽度、适应高度或实际尺寸。
- 同步状态：明细默认开启；坏例单图不展示该选项。
- 缩放倍率，限制在 `0.1` 到 `12` 之间。
- 归一化观察中心，用于不同分辨率图片之间同步平移。
- 放大镜、背景主题、当前活动图片和临时拖拽状态。

支持以下操作：

- 鼠标滚轮以指针所在位置为中心缩放。
- 直接拖动图片平移；按住空格可临时进入平移模式。
- `+` 和 `-` 调整缩放，`Esc` 关闭高清预览。
- 工具栏可以切换四种显示模式、放大镜和深浅背景，也可以复位预览。
- 明细中的覆盖对比按钮仅在按住期间将源图覆盖到目标图，松开指针、键盘、窗口失焦或关闭弹窗时立即清理。
- 输入框、下拉框和文本区域聚焦时不响应预览快捷键。

## 数据流

1. 明细缩略图被点击时，现有行数据构造成双图或三图预览载荷。
2. 坏例缩略图被点击时，仅构造包含当前模型图片的单图预览载荷。
3. 打开覆盖层时创建新的预览组并渲染对应图片舞台、工具栏和可用的覆盖对比按钮。
4. 图片尺寸可用后注册到控制器，并应用默认的适应窗口、居中状态。
5. 用户操作更新控制器状态；明细同步开启时将倍率和归一化中心传播给其他有效图片。
6. 关闭覆盖层时释放指针捕获、事件监听、放大镜和临时覆盖图层，并重置覆盖层内容。

## 错误与边界处理

- 参考图为空时不渲染空白参考图舞台。
- 单张图片加载失败不会导致覆盖层或关闭操作失效。
- 已失败图片不参与同步尺寸和变换计算。
- 缩放和平移受限，避免图片完全移出可视区域或出现无穷变换值。
- 重复快速打开不同记录时，旧图片的异步加载回调不得修改新预览组。
- 覆盖层关闭动作不得连带关闭其下方的明细或坏例筛选弹窗。
- 打开新预览时始终恢复默认模式、居中位置、深色背景；明细同步默认开启。

## 测试方案

### 自动测试

- 新增看板高清预览前端契约测试，验证控制器、预览舞台、工具栏和关闭入口存在。
- 验证明细载荷按 T2I/TI2I 渲染两图或三图，并包含实际模型标签。
- 验证坏例载荷只渲染单图，且不包含同步开关和覆盖对比按钮。
- 使用 Node 运行时探针验证缩放上下限、归一化中心同步、关闭同步后的独立变换和复位行为。
- 验证滚轮、指针拖拽、快捷键守卫、加载失败与关闭清理钩子。
- 运行完整 Python 测试套件，确认看板筛选、权限、导出和评测界面预览不回归。

### 手动验证

- 分别打开 T2I 明细双图、TI2I 明细三图和坏例单图。
- 验证明细同步缩放、同步平移、解除同步和按住覆盖对比。
- 验证坏例只有单图且没有同步或对比按钮。
- 验证四种显示模式、放大镜、背景切换、复位、滚轮、拖动及快捷键。
- 验证窄屏布局、图片加载失败、快速切换记录和关闭后重新打开。

## 完成标准

- 明细高清预览的视觉布局和图片检查能力与评测界面一致，并保留完整双图/三图覆盖对比。
- 坏例高清预览保持单图，同时具备除同步和图间对比外的评测高清预览工具。
- 所有图片保持原始宽高比，可以自由缩放和平移并可靠复位。
- 打开、关闭和重复切换预览不会泄漏临时事件或残留上一条记录的状态。
- 现有后端接口和看板其他功能保持不变，完整测试套件通过。
